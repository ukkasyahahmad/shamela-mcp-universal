# shamela-mcp

هذا الدليل موجَّه إلى **المستخدم العادي**، وخصوصًا مستخدم **المكتبة الشاملة** الذي يريد استعمال مكتبته المحليَّة من داخل تطبيق ذكاء اصطناعي يدعم MCP.

## ما هذا؟

`shamela-mcp` خادم MCP محلِّي يربط تطبيق الذكاء الاصطناعي لديك بتثبيت **المكتبة الشاملة ٤** الموجود على جهازك.

وبذلك يستطيع التطبيق:

- البحث في الكتب التي نزَّلتها داخل الشاملة
- البحث في المؤلفين وعناوين الكتب والآيات
- قراءة صفحات الكتب
- قراءة الفهارس والأبواب
- إنشاء إحالات واقتباسات من صفحات الكتب

كلُّ ذلك يتمُّ **محلِّيًّا على جهازك**. والخادم لا يكتب في قاعدة بيانات الشاملة.

## لمن هذا؟

يناسبك إذا:

- كنت تستعمل المكتبة الشاملة ٤ بالفعل
- نزَّلت كتابًا واحدًا على الأقل داخل الشاملة
- أردت أن تسأل تطبيقًا ذكيًّا اعتمادًا على مكتبتك المحليَّة

## ما الذي تحتاجه؟

- المكتبة الشاملة ٤ مثبَّتة على جهازك
- كتاب واحد على الأقل منزَّل داخل الشاملة
- تطبيق ذكاء اصطناعي يدعم **MCP المحلي عبر stdio**
- Node.js 20 أو أحدث

## طريقة الاستعمال للمستخدم العادي

### ١) نزِّل الإصدار الجاهز

نزِّل آخر إصدار من صفحة Releases في GitHub.

اختر أحد الملفَّين:

- `shamela-mcp-<version>.zip`
- `shamela-mcp-<version>.tgz`

في الغالب، إن كنت على ويندوز فاختر ملف `.zip`.

### ٢) فكَّ الضغط

فكَّ الضغط في مجلد يسهل الرجوع إليه.

مثلًا:

- `C:\tools\shamela-mcp`
- `/Users/yourname/tools/shamela-mcp`

بعد الفك، يفترض أن ترى:

- `dist/index.js`
- `helper/shamela-helper.jar`
- `examples/`
- `docs/`

### ٣) أضف إعداد MCP في تطبيقك

أبسط إعداد هو:

```json
{
  "mcpServers": {
    "shamela": {
      "command": "node",
      "args": ["/ABSOLUTE/PATH/TO/shamela-mcp/dist/index.js"]
    }
  }
}
```

استبدل المسار بمسارك الحقيقي على جهازك.

مثال على ويندوز:

```json
{
  "mcpServers": {
    "shamela": {
      "command": "node",
      "args": ["C:\\tools\\shamela-mcp\\dist\\index.js"]
    }
  }
}
```

يوجد مثال جاهز في [../examples/universal-mcp.json](../examples/universal-mcp.json).

### ٤) أغلِق التطبيق وافتحه من جديد

بعد حفظ إعداد MCP، أغلِق التطبيق إغلاقًا كاملًا ثم افتحه من جديد.

### ٥) جرِّب أوَّل سؤال

مثال:

- `ابحث في مكتبتي الشاملة عن كلمة "الكلام"`
- `افتح الصفحة التي تشرح تعريف الكلام`
- `ما الكتب الموجودة عندي لابن عثيمين؟`

إذا كان كلُّ شيء صحيحًا، فسيبدأ التطبيق باستعمال أدوات `shamela_*`.

## هل يجب أن أكتب مسار الشاملة وجافا يدويًّا؟

في الغالب **لا**.

الخادم يحاول تلقائيًّا:

- العثور على مجلد الشاملة
- العثور على جافا المرفقة مع الشاملة

إذا فشل الاكتشاف التلقائي، عندها فقط استخدم متغيِّرات `env`.

انظر المثال: [../examples/universal-mcp-with-env.json](../examples/universal-mcp-with-env.json)

## إذا فشل الاكتشاف التلقائي

استخدم إعدادًا مثل هذا:

```json
{
  "mcpServers": {
    "shamela": {
      "command": "node",
      "args": ["C:\\tools\\shamela-mcp\\dist\\index.js"],
      "env": {
        "SHAMELA_INSTALL_ROOT": "C:\\path\\to\\Shamela",
        "SHAMELA_JRE": "C:\\path\\to\\java.exe"
      }
    }
  }
}
```

المسار `SHAMELA_INSTALL_ROOT` هو المجلد الذي يحتوي على:

- `database`
- `app`

## أمثلة لأسئلة واقعيَّة

- `ابحث عن الاستصناع في الكتب الفقهية التي نزَّلتها`
- `افتح الصفحة التي فيها شرح القياس`
- `اعرض لي تفسير آية الكرسي من كتب التفسير الموجودة عندي`
- `أنشئ لي إحالة من هذه الصفحة`

## مشكلات شائعة

### الأدوات لا تظهر

الأسباب المعتادة:

- إعداد MCP غير صحيح
- مسار `dist/index.js` غير صحيح
- التطبيق لم يُعَد تشغيله

### لم يُعثر على الشاملة

السبب المعتاد:

- تثبيت الشاملة في مسار غير شائع

الحل:

- اضبط `SHAMELA_INSTALL_ROOT` يدويًّا

### لم يُعثر على جافا

الحل:

- اضبط `SHAMELA_JRE` يدويًّا

### لا تظهر نتائج

تحقَّق من:

- هل الكتاب منزَّل فعلًا داخل الشاملة؟
- هل عبارة البحث صحيحة؟
- هل قيَّدت البحث بنطاق ضيِّق جدًّا؟

## للمستخدم التقني

إن أردت البناء من المصدر:

```bash
npm install
npm run build
node dist/index.js
```

لكن للمستخدم العادي، الأفضل أن يستعمل **الإصدار الجاهز من GitHub** بدل البناء اليدوي.
